## 当前实现与目标基线的差异

- 当前实现：
  已进入“有状态会话版”阶段，但当前仍然只覆盖：
  - 纯 LLM 同步/流式聊天
  - conversation/message/trace 持久化
  - 官方 `TokenWindowChatMemory / MessageWindowChatMemory + ChatMemoryStore(MySQL)` 上下文管理
  - 第二轮统一编排骨架：`ChatOrchestrator + ModeResolver + ChatExecutionSpec + ChatOutputPort + TracePublisher + RagRuntimeResolver + ToolRuntimeResolver`
  - 最小会话串行保护
  - 前端会话列表与 Trace 时间线
  - 前端聊天前台与后台管理区路由分层
  - 聊天页“侧边栏 + 对话主区”骨架
  - Trace 主区内摘要块与配置抽屉入口
  - 知识库创建 / 列表
  - 知识库文件上传 / 列表
  - 知识库文件索引
  - 知识库测试检索
  - 聊天页会话级 `memory / KB / MCP` 默认配置
  - 聊天请求级 `knowledgeBaseIds / mcpServerIds` 覆盖语义
  - 前端知识库文件级管理与检索测试
  - Embedding 与 Milvus 基础配置装配
  - LOCAL 文件存储落盘
  - Tika 文档解析
  - 段落优先基础分片
  - `knowledge_segment` 入库与 Milvus 向量写入
  - 按 partition 的 Milvus 检索 + MySQL 原文回查
  - LangChain4j 官方 `DefaultRetrievalAugmentor + EmbeddingStoreContentRetriever` RAG 注入
  - 第一版 Prompt 模板目录与非 Agent 主链路 Prompt 装配
  - `RagRuntimeResolver` 对统一 Prompt 目录的复用
  - Agent 路径的统一 system prompt 注入
  - Prompt 组成摘要 Trace 与调试视图
  - 创建知识库时自动创建/校验 Milvus partition
  - MCP Server 管理页与工具快照管理
  - LangChain4j 官方 MCP client 的连接测试与工具同步
  - 聊天主链路中的 MCP Tool 动态装配

- 目标基线：
