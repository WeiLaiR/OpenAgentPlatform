# 当前实现清单

## 已验证通过

### 后端

- 已补充统一响应模型 `ApiResponse<T>`。
- 已补充统一错误模型 `ApiError`。
- 已实现请求级 `requestId` 生成与响应头透传，当前头名为 `X-Request-Id`。
- 已实现全局异常处理，当前覆盖：
  - `IllegalArgumentException`
  - `MethodArgumentNotValidException`
  - 会话不存在
  - 同会话并发请求冲突
  - 其他未捕获异常
- 已实现 `GET /api/v1/system/health` 健康检查接口。
- 已接入 LangChain4j OpenAI-compatible ChatModel / StreamingChatModel 的条件装配。
- 已接入 LangChain4j OpenAI-compatible EmbeddingModel 的条件装配。
- 已接入 MilvusClient 条件装配，固定当前默认 `database=openagent`、`collection=knowledge_segment`。
- 已实现 `POST /api/v1/chat/send-sync` 同步聊天接口。
- 已实现 `POST /api/v1/chat/send` + `GET /api/v1/chat/stream/{requestId}` 两步式流式聊天接口。
- 已实现流式聊天请求的内存态 session 缓存与 SSE 事件回放。
- 已实现 `GET /api/v1/traces/{requestId}` Trace 明细接口。
- 已实现 `GET /api/v1/traces/stream/{requestId}` Trace SSE 接口。
- 已实现 `POST /api/v1/conversations` 创建会话接口。
- 已实现 `GET /api/v1/conversations` 会话列表接口。
- 已实现 `GET /api/v1/conversations/{conversationId}/messages` 历史消息查询接口。
- 已实现 `PUT /api/v1/conversations/{conversationId}/settings` 会话模式开关更新接口，当前支持 `enableRag / enableAgent` 持久化。
- 已实现 `POST /api/v1/knowledge-bases` 创建知识库接口。
- 已实现 `GET /api/v1/knowledge-bases` 知识库列表接口。
- 已实现 `POST /api/v1/knowledge-bases/{kbId}/files/upload` 文件上传接口。
- 已实现 `GET /api/v1/knowledge-bases/{kbId}/files` 文件列表接口。
- 已实现 `POST /api/v1/knowledge-files/{fileId}/index` 文件索引接口。
- 已实现 `POST /api/v1/knowledge-bases/retrieve` 知识库检索接口。
- 已实现 `GET /api/v1/mcp/servers` / `POST /api/v1/mcp/servers` / `PUT /api/v1/mcp/servers/{serverId}` MCP Server 管理接口。
- 已实现 `POST /api/v1/mcp/servers/{serverId}/connect` 与 `POST /api/v1/mcp/servers/{serverId}/sync-tools`，并通过 LangChain4j 官方 MCP client 完成连通性测试与工具快照同步。
- 已实现 `GET /api/v1/mcp/tools` 与 `POST /api/v1/mcp/tools/{toolId}/enable` / `POST /api/v1/mcp/tools/{toolId}/disable` 工具快照管理接口。
- 已实现聊天请求携带 `knowledgeBaseIds` 时的 RAG 检索与上下文注入。
- 已实现聊天请求自动解析/创建 `conversation`，并持久化 `conversation_message`。
- 已正式接入 LangChain4j 官方 `ChatMemory` 主线能力：优先使用 `TokenWindowChatMemory`；当官方 `TokenCountEstimator` 无法识别当前模型时，自动降级到官方 `MessageWindowChatMemory`，避免非 OpenAI 模型启动失败。
- 已支持通过 `openagent.ai.chat.tokenizer-model-name` 为 OpenAI-compatible 非 OpenAI 模型显式指定 tokenizer 参考模型名，以继续启用 token window。
- 已实现自定义 `ChatMemoryStore(MySQL)`，将当前 memory 视图持久化到 `chat_memory_session` / `chat_memory_message`。
- 已实现 `conversationId -> memoryId` 的稳定映射，当前直接以 `conversationId` 作为 LangChain4j ChatMemory id。
- 已实现 memory 与 history 分离：`conversation_message` 继续保存完整历史，`chat_memory_message` 只保存当前模型上下文窗口。
- 已实现“请求成功后才回写 ChatMemory”的更新策略，避免失败轮次提前污染后续上下文。
- 已实现知识库创建时自动生成 `kb_{knowledgeBaseId}` 分区名，并创建/校验对应 Milvus partition。
- 已实现 LOCAL 模式下的知识库文件落盘与 `file_hash` / `storage_uri` 入库。
- 已实现基于 Tika 的 txt / md / docx 文档解析。
- 已实现段落优先 + `chunkSize` / `chunkOverlap` 的基础分片。
- 已实现 `knowledge_segment` 原文片段入库与 `milvus_primary_key` 生成。
- 已实现基于 LangChain4j `EmbeddingModel` 的分片向量生成。
- 已实现知识库文件索引时的 Milvus 向量写入与失败状态回写。
- 已实现按 `knowledgeBaseIds -> partition` 的 Milvus 检索与 MySQL 原文片段回查。
- 已实现聊天主链路中的 RAG 片段注入与 `RAG_RETRIEVAL_*` Trace 事件。
- 已实现 `enableAgent` 打开时的 MCP Tool 动态装配，基于 LangChain4j 官方 `McpToolProvider` / `ToolExecutor` 完成 tool loop 与结果回填。
- 已实现 `trace_event` 入库，当前覆盖：
  - `USER_MESSAGE_RECEIVED`
  - `MODEL_REQUEST_STARTED`
  - `FINAL_RESPONSE_STREAMING`
  - `FINAL_RESPONSE_COMPLETED`
  - `ERROR_OCCURRED`
- 已实现 conversation 维度的最小串行保护，避免同会话并发生成。
- 已补充基础 CORS 配置，允许本地 Vite 开发端口访问 `/api/**`。
- 未配置模型地址时，后端仍可正常启动，聊天接口会返回“模型未配置”的明确错误。

## 当前实现与目标基线的差异

- 当前实现：
  已进入“有状态会话版”阶段，但当前仍然只覆盖：
  - 纯 LLM 同步/流式聊天
  - conversation/message/trace 持久化
  - 官方 `TokenWindowChatMemory / MessageWindowChatMemory + ChatMemoryStore(MySQL)` 上下文管理
  - 最小会话串行保护
  - 前端会话列表与 Trace 时间线
  - 前端系统页 / 对话页 / 知识库页三页拆分
  - 知识库创建 / 列表
  - 知识库文件上传 / 列表
  - 知识库文件索引
  - 知识库测试检索
  - 聊天请求级 `knowledgeBaseIds`
  - 前端知识库文件级管理与检索测试
  - Embedding 与 Milvus 基础配置装配
  - LOCAL 文件存储落盘
  - Tika 文档解析
  - 段落优先基础分片
  - `knowledge_segment` 入库与 Milvus 向量写入
  - 按 partition 的 Milvus 检索 + MySQL 原文回查
  - 聊天主链路中的 RAG 上下文注入
  - 创建知识库时自动创建/校验 Milvus partition
  - MCP Server 管理页与工具快照管理
  - LangChain4j 官方 MCP client 的连接测试与工具同步
  - 聊天主链路中的 MCP Tool 动态装配

- 目标基线：
  后续还需继续接上：
  - 前端片段级编辑与向量重算联动
  - Agent 模式与 RAG + Agent 联合编排
  - 更完整的 LangChain4j observability 接入
