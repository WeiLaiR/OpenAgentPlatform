# 当前实现清单

更新日期：2026-03-16

## 已验证通过

### 后端

- 已补充统一响应模型 `ApiResponse<T>`。
- 已补充统一错误模型 `ApiError`。
- 已实现请求级 `requestId` 生成与响应头透传，当前头名为 `X-Request-Id`。
- 已实现全局异常处理，当前覆盖：
  - `IllegalArgumentException`
  - `MethodArgumentNotValidException`
  - 会话不存在
  - 同会话并发请求冲突
  - 其他未捕获异常
- 已实现 `GET /api/v1/system/health` 健康检查接口。
- 已接入 LangChain4j OpenAI-compatible ChatModel / StreamingChatModel 的条件装配。
- 已实现 `POST /api/v1/chat/send-sync` 同步聊天接口。
- 已实现 `POST /api/v1/chat/send` + `GET /api/v1/chat/stream/{requestId}` 两步式流式聊天接口。
- 已实现流式聊天请求的内存态 session 缓存与 SSE 事件回放。
- 已实现 `GET /api/v1/traces/{requestId}` Trace 明细接口。
- 已实现 `GET /api/v1/traces/stream/{requestId}` Trace SSE 接口。
- 已实现 `POST /api/v1/conversations` 创建会话接口。
- 已实现 `GET /api/v1/conversations` 会话列表接口。
- 已实现 `GET /api/v1/conversations/{conversationId}/messages` 历史消息查询接口。
- 已实现聊天请求自动解析/创建 `conversation`，并持久化 `conversation_message`。
- 已实现初版上下文管理：固定回灌最近 5 轮已完成的 `USER` / `ASSISTANT` 正式消息，提供基础连续对话能力。
- 已实现 `trace_event` 入库，当前覆盖：
  - `USER_MESSAGE_RECEIVED`
  - `MODEL_REQUEST_STARTED`
  - `FINAL_RESPONSE_STREAMING`
  - `FINAL_RESPONSE_COMPLETED`
  - `ERROR_OCCURRED`
- 已实现 conversation 维度的最小串行保护，避免同会话并发生成。
- 已补充基础 CORS 配置，允许本地 Vite 开发端口访问 `/api/**`。
- 未配置模型地址时，后端仍可正常启动，聊天接口会返回“模型未配置”的明确错误。

### 前端

- 已接入 `Element Plus`。
- 已建立基础路由首页 `/`。
- 已建立统一请求封装。
- 已接入后端健康检查接口并在首页展示状态。
- 已补充会话列表面板，支持新建会话与切换会话。
- 已补充聊天主面板，支持同步发送与流式发送。
- 已补充历史消息回放，刷新后可重新加载会话消息。
- 已补充 Trace 面板，支持展示请求级时间线。
- 已接入浏览器 `EventSource` 订阅 `/api/v1/chat/stream/{requestId}`。
- 已接入浏览器 `EventSource` 订阅 `/api/v1/traces/stream/{requestId}`。
- 已补充 Vite 开发代理，`/api` 默认转发到 `http://localhost:8080`。

## 当前实现与目标基线的差异

- 当前实现：
  已进入“有状态会话版”阶段，但当前仍然只覆盖：
  - 纯 LLM 同步/流式聊天
  - conversation/message/trace 持久化
  - 最近 5 轮消息的初版手工上下文回灌
  - 最小会话串行保护
  - 前端会话列表与 Trace 时间线

- 目标基线：
  后续还需继续接上：
  - RAG 检索与知识库管理
  - MCP Server 管理与工具快照
  - Agent 模式与 RAG + Agent 联合编排
  - 完整的 ChatMemory / ChatMemoryStore 上下文管理
  - 更完整的 LangChain4j observability 接入

## 当前可作为下一步起点的内容

- 进入下一阶段时，可以直接在当前会话主链路上接 RAG：
  - `knowledge_base`
  - `knowledge_file`
  - `knowledge_segment`
  - 文件上传 / 索引 / 检索
  - 聊天接口中的 `enableRag`
- 在 RAG 跑通后，再接 MCP 与 Agent：
  - `mcp_server`
  - `mcp_tool_snapshot`
  - Tool 动态装配
  - `enableAgent`
