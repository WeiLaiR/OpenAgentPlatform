# 实现状态文档目录

该目录用于存放与“当前实现情况”和“开发推进状态”相关的 Markdown 文档，和 `docs/` 中的设计方案、架构说明分开维护。

建议放置的文档类型：

- `当前实现清单.md`：记录前端、后端已经落地的功能与接口。
- `当前优先实现清单.md`：基于当前实现与 `docs/` 目标基线的差异，记录现阶段最应该优先补齐的实现项。
- `开发进度记录.md`：按日期或阶段记录开发进展。
- `待办与阻塞项.md`：跟踪未完成事项、风险和依赖问题。

建议约定：

- 只记录当前实际状态，不写长期设计方案。
- 只在“已经实现且验证通过”后补充内容，不提前登记计划中的功能。
- `当前优先实现清单.md` 不写长期路线图，只写“基于当前实现状态判断出来的现阶段优先项”。
- 每次完成功能或模块后，先更新 `当前实现清单.md`，再重新分析与 `docs/` 的差距，随后同步更新 `当前优先实现清单.md`。
- 文档标题尽量直接，例如“聊天模块当前实现情况”。
- 涉及代码或接口变更时，补充对应模块路径，例如 `frontend/src/`、`backend/src/main/java/`。
