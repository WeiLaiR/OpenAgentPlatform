# Open Agent Platform (Java) 配置清单文档

## 1. 文档目标

本文档用于梳理项目运行所需的核心配置项，方便你在开发、联调、排错时快速定位配置来源。

目标：

- 明确必备配置项
- 约定配置分层
- 减少后期配置混乱
- 为多环境扩展预留空间

原则：

- 先满足本地开发
- 敏感信息尽量不写死
- 配置按模块归类
- 能外置的尽量外置

------

## 2. 配置分层建议

建议至少分为以下几层：

### 2.1 应用基础配置

例如：

- 服务端口
- 应用名称
- 日志级别
- 时区
- 文件上传大小限制

### 2.2 数据源配置

例如：

- MySQL
- Milvus

### 2.3 AI 模型配置

例如：

- Chat Model
- Embedding Model

### 2.4 RAG 配置

例如：

- 文档解析
- 分块参数
- 检索参数

### 2.5 MCP 配置

例如：

- 默认连接策略
- 超时
- 工具同步策略

### 2.6 流式输出与前端联调配置

例如：

- SSE 超时
- 跨域
- 前端 base URL

------

## 3. 环境划分建议

建议至少准备 3 套环境概念：

- `local`：本地开发
- `test`：联调/测试
- `prod`：正式环境

当前阶段你只需要优先维护好：

- `application.yml`
- `application-local.yml`

就够了。

------

## 4. 后端核心配置清单

------

## 4.1 应用基础配置

### 必配项

- `server.port`
- `spring.application.name`
- `spring.profiles.active`
- `spring.servlet.multipart.max-file-size`
- `spring.servlet.multipart.max-request-size`

### 建议项

- 应用时区
- Jackson 时间序列化格式
- 全局编码 UTF-8
- 错误栈是否返回前端

### 说明

你这个项目后期有文件上传、SSE、长链路请求，所以文件大小和超时类配置最好尽早明确。

------

## 4.2 MySQL 配置

### 必配项

- `spring.datasource.url`
- `spring.datasource.username`
- `spring.datasource.password`
- `spring.datasource.driver-class-name`

### 建议项

- 连接池大小
- 最小空闲连接
- 连接超时
- SQL 日志打印

### 当前建议

开发环境先保证：

- 能正常连库
- 能建表
- 能打印 SQL

先别过度优化连接池参数。

------

## 4.3 Milvus 配置

### 必配项

- `milvus.host`
- `milvus.port`

### 说明

你当前用的是 Milvus standalone，Java 客户端主要连的是 **19530**，不是 MinIO 端口。
 MinIO 只是 Milvus 内部对象存储，不是业务侧直接访问入口。

### 建议项

- 默认 collection 名称
- 默认 database 名称
- 连接超时
- 检索超时

### 当前建议

先只保证：

- 能连通 Milvus
- 能正常创建 collection / partition
- 能写入和检索向量

------

## 4.4 Chat Model 配置

### 必配项

- `ai.chat.base-url`
- `ai.chat.api-key`
- `ai.chat.model-name`

### 建议项

- `temperature`
- `top-p`
- `max-tokens`
- 请求超时
- 重试次数

### 当前场景

你现在的 Chat Model 是本地通过 llama server 部署的 qwen3.5 32b-a3b，所以只要它兼容 OpenAI API 风格，后端就可以按 OpenAI-compatible 模式配置。

### 当前建议

第一版先不要暴露太多模型参数到前端，先在配置文件中写默认值即可。

------

## 4.5 Embedding Model 配置

### 必配项

- `ai.embedding.base-url`
- `ai.embedding.api-key`
- `ai.embedding.model-name`

### 建议项

- 向量维度
- 请求超时
- batch 大小

### 当前场景

你本地已经部署了适配 OpenAI API 的 bge-m3，所以这部分和 chat model 很类似。

### 特别注意

Embedding 维度必须和知识库、Milvus collection 设计一致。
 后面如果换 embedding 模型，不要直接混用旧 collection。

------

## 4.6 LangChain4j 配置

### 建议配置项

- chat model bean 相关参数
- streaming model bean 相关参数
- embedding model bean 相关参数
- chat memory 默认策略
- token window 大小

### 当前建议

第一版不必把所有 LangChain4j 参数都外露成配置项。
 先把以下内容配置化即可：

- chat model
- embedding model
- memory 默认开关
- memory window 大小

------

## 4.7 RAG 配置

### 必配项

- 默认 `topK`
- 最低相似度阈值 `minScore`
- 默认 chunk size
- 默认 chunk overlap

### 建议项

- 是否返回检索片段
- 每次最多注入多少个 segment
- 检索结果最大 token 限制

### 当前建议

第一版使用固定默认值即可，例如：

- `topK = 3~5`
- `minScore = 0.6~0.75`
- `chunkSize = 800~1200`
- `chunkOverlap = 100~200`

先跑通，再调优。

------

## 4.8 文档解析配置

### 建议配置项

- 默认 parser 类型
- 支持的文件扩展名白名单
- 单文件最大大小
- 是否保留原文
- 是否记录解析中间结果

### 当前建议

第一版先支持：

- pdf
- docx
- md
- txt

不要一开始就把所有格式都打开。

------

## 4.9 MCP 配置

### 必配项

- MCP 默认协议类型
- MCP 默认超时
- Tool 同步超时
- Tool 调用超时

### 建议项

- 是否自动同步工具
- 是否允许高风险工具默认启用
- 是否允许会话自动继承所有可用 MCP

### 当前建议

第一版保持保守：

- 新增 MCP 后需要手动连接
- 手动同步工具
- 高风险工具默认不自动启用

------

## 4.10 Memory 配置

### 建议配置项

- 默认 memory 是否开启
- memory 类型
- token window 大小
- message window 大小
- memory 持久化开关

### 当前建议

第一版直接定：

- 开启 memory
- 使用 `TokenWindow`
- 配一个默认 `maxTokens`

后面稳定后再调整。

------

## 4.11 SSE 与异步配置

### 必配项

- SSE 超时时间
- 异步线程池大小
- 请求超时时间

### 说明

你的平台有：

- 流式输出
- 文档索引
- MCP 调用
- 可能的异步任务

所以异步执行器和 SSE 超时最好尽早独立配置。

### 当前建议

第一版只要保证：

- SSE 不会太快断开
- 后台索引不会阻塞主线程

------

## 4.12 日志配置

### 必配项

- root 日志级别
- 应用包日志级别

### 建议项

- SQL 日志
- Milvus 相关日志
- LangChain4j 请求/响应日志
- MCP 连接日志

### 当前建议

开发环境可以稍微详细一点，但不要把模型完整 prompt 和敏感参数无脑全打出来。

------

## 4.13 跨域配置

### 前后端分离开发必配

- 允许的前端 origin
- 允许的方法
- 允许的 headers
- 是否允许 credentials

### 当前建议

本地开发阶段允许：

- 前端本地地址访问后端
- SSE 路径同样允许跨域

------

## 5. 前端配置清单

------

## 5.1 基础运行配置

### 必配项

- `VITE_APP_TITLE`
- `VITE_API_BASE_URL`
- `VITE_SSE_BASE_URL`

### 说明

前端要至少区分：

- 普通 HTTP API 地址
- SSE 地址

如果前后端同域部署，后面可以统一；本地开发阶段一般还是分开更清晰。

------

## 5.2 页面行为配置

### 建议项

- 默认是否开启 RAG
- 默认是否开启 Agent
- 默认展开 Trace 面板
- 默认选中的知识库
- 默认选中的 MCP server

### 当前建议

这类配置先放前端常量或 store，不急着做后端动态配置中心。

------

## 5.3 上传配置

### 建议项

- 支持文件类型白名单
- 单文件大小提示
- 上传失败提示文案
- 索引状态轮询间隔

------

## 5.4 调试配置

### 建议项

- 是否显示 debug 面板
- 是否打印 SSE 原始事件
- 是否显示 requestId

### 当前建议

开发期建议打开 requestId 和基础调试输出，后期可关闭。

------

## 6. 敏感信息管理建议

以下信息不要直接硬编码在代码里：

- 数据库密码
- 模型 API key
- MCP 认证信息
- 自定义 headers 中的 token
- 远程服务凭证

### 当前建议

开发阶段至少做到：

- 放在 `application-local.yml`
- 或放在环境变量中引用

不要把这些值直接写进 Java 代码或提交到公共仓库。

------

## 7. 建议的配置文件组织方式

后端建议：

- `application.yml`
   放公共默认配置
- `application-local.yml`
   放本地开发配置
- `application-test.yml`
   后续联调用
- `application-prod.yml`
   后续部署再补

前端建议：

- `.env`
- `.env.development`
- `.env.production`

------

## 8. 当前阶段的最小必配集

如果你现在只想尽快进入开发，下面这些是最小必要配置：

### 后端

- 服务端口
- MySQL 连接
- Milvus 地址
- Chat Model base-url / api-key / model-name
- Embedding Model base-url / api-key / model-name
- 文件上传大小
- SSE 超时
- 跨域配置

### 前端

- API base URL
- SSE base URL

只要这些配好，你就可以开始做：

- 基础聊天
- 会话存储
- 文件上传
- RAG 检索
- MCP 管理

------

## 9. 当前不需要过度细化的配置

这些先不用花太多时间：

- 多环境自动切换脚本
- 配置中心
- 加密配置管理平台
- 动态热更新配置
- 复杂限流参数
- 复杂缓存参数
- 多租户隔离参数

因为你现在最重要的是先把功能做出来。

------

## 10. 一句话总结

当前配置策略建议是：

**先把运行必需项配齐，把敏感信息外置，把 AI / RAG / MCP / SSE 四类配置独立出来，其余高级配置先不展开。**
